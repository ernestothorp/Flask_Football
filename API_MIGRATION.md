# API Migration Guide - API-Football.com

## Overview

The project has been migrated from web scraping the DFB website to using the API-Football API (via RapidAPI). The DFB website changed its structure making the old scraping approach obsolete.

## Setup Instructions

### 1. Get an API Key

1. Go to [RapidAPI API-Football](https://rapidapi.com/api-sports/api/api-football)
2. Sign up for a free account
3. Subscribe to the API-Football API (Free tier available: 100 requests/day)
4. Copy your API key from the dashboard

### 2. Configure API Key

Set your API key as an environment variable:

**Linux/Mac:**
```bash
export API_FOOTBALL_KEY="your_api_key_here"
```

**Windows (Command Prompt):**
```cmd
set API_FOOTBALL_KEY=your_api_key_here
```

**Windows (PowerShell):**
```powershell
$env:API_FOOTBALL_KEY="your_api_key_here"
```

Alternatively, edit `GET_Data_API.py` and replace `YOUR_API_KEY_HERE` with your actual key:
```python
API_KEY = os.getenv("API_FOOTBALL_KEY", "YOUR_ACTUAL_API_KEY")
```

## Usage

### Fetch League Standings

```python
from GET_Data_API import GET_TABLE

league_name = "1_Bundesliga"  # Options: "1_Bundesliga", "2_Bundesliga", "3_Liga"
season = 2024

standings = GET_TABLE(league_name, season)
print(standings)
```

### Fetch Fixtures for a Specific Round

```python
from GET_Data_API import CREATE_CSV

league_name = "1_Bundesliga"
season = 2024
round_number = 13

CREATE_CSV(league_name, season, round_number)
```

### Fetch All Rounds for a Season

```python
from GET_Data_API import GET_ALL_ROUNDS

league_name = "1_Bundesliga"
season = 2024

GET_ALL_ROUNDS(league_name, season, max_rounds=34)
```

## CSV Output Format

The CSV files maintain the same format as before:

```csv
Spieltag,Status,Team_1,Team_2,Tore_Team_1,Tore_Team_2
01,PASS,Bayern München,Borussia Dortmund,3,1
02,OPEN,RB Leipzig,Bayern München,,
```

- **Spieltag**: Match day/round number
- **Status**: PASS (completed) or OPEN (not played yet)
- **Team_1**: Home team
- **Team_2**: Away team
- **Tore_Team_1**: Goals scored by home team
- **Tore_Team_2**: Goals scored by away team

## API Rate Limits

**Free Tier:**
- 100 requests per day
- Good for testing and small projects

**Pro Tiers:**
- Higher request limits available
- See [API-Football Pricing](https://rapidapi.com/api-sports/api/api-football/pricing)

## Tips

1. **Cache Data**: Save API responses to avoid hitting rate limits
2. **Update Frequency**: Football data doesn't change constantly; update once per day or after match days
3. **Error Handling**: The API returns proper error codes; check responses before processing

## Troubleshooting

### "403 Forbidden" Error
- Check that your API key is correct
- Verify you haven't exceeded rate limits
- Ensure your RapidAPI subscription is active

### "No data available"
- Verify the season year is correct (e.g., 2024, not 2024/2025)
- Check that the league ID is correct
- Some historical data may not be available

### Empty CSV Files
- Check API response in console
- Verify fixtures exist for the specified round
- Ensure season parameter matches current/past seasons

The old `GET_Data.py` file is kept for reference but is no longer functional with the current DFB website. All new development should use `GET_Data_API.py`.

The `GET_Calc.py` module remains unchanged and works with CSV files generated by either the old or new system.
